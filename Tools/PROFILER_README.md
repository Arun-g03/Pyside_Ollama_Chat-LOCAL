# Application Profiler Tool

A comprehensive profiling tool for the PySide Ollama Chat application that monitors system resources, application performance, and generates detailed reports.

## Features

- **System Resource Monitoring**: CPU, memory, disk I/O, and network usage
- **Application Performance Profiling**: Function-level profiling using cProfile
- **Process Monitoring**: Main process and child process information
- **Real-time Data Collection**: Continuous monitoring during application execution
- **Comprehensive Reports**: JSON and human-readable summary reports
- **Flexible Duration**: Configurable profiling duration

## Requirements

- Python 3.7+
- PySide6
- psutil
- cProfile (built-in)

## Installation

The profiler uses standard Python libraries. Make sure you have the required dependencies:

```bash
pip install psutil PySide6
```

## Usage

### Basic Usage

Run the profiler with default settings (60 seconds):

```bash
python Tools/run_profiler.py
```

### Advanced Usage

```bash
# Profile for 120 seconds
python Tools/run_profiler.py --duration 120

# Quick profiling mode (30 seconds)
python Tools/run_profiler.py --quick

# Custom output directory
python Tools/run_profiler.py --output-dir my_reports

# Combine options
python Tools/run_profiler.py --duration 300 --output-dir detailed_reports
```

### Direct Usage

You can also run the profiler directly:

```bash
python Tools/profiler.py --duration 60 --output-dir profiling_reports
```

## Command Line Options

- `--duration`: Profiling duration in seconds (default: 60)
- `--output-dir`: Output directory for reports (default: profiling_reports)
- `--quick`: Quick profiling mode (30 seconds)

## Output

The profiler generates two types of reports:

### 1. JSON Report (`profiling_report_YYYYMMDD_HHMMSS.json`)

Contains detailed profiling data including:
- System information (CPU, memory, disk, network)
- Monitoring data points with timestamps
- Process information (main and child processes)
- Function-level profiling statistics
- Performance summaries

### 2. Summary Report (`profiling_report_YYYYMMDD_HHMMSS_summary.txt`)

Human-readable summary including:
- Profiling duration
- System resource usage (CPU, memory)
- Process performance metrics
- Top functions by execution time
- Performance recommendations

## Use Cases

### Performance Analysis
- Identify CPU and memory bottlenecks
- Monitor application resource usage over time
- Analyze function performance

### System Monitoring
- Track system resource utilization
- Monitor child process behavior
- Analyze I/O patterns

### Development Optimization
- Profile specific features
- Compare performance before/after changes
- Identify optimization opportunities

## Tips

1. **Duration**: Use longer durations (300+ seconds) for comprehensive analysis
2. **Quick Mode**: Use `--quick` for rapid testing (30 seconds)
3. **Output Directory**: Organize reports by feature or date
4. **Multiple Runs**: Run multiple profiles to get average performance metrics
5. **Compare Reports**: Use the JSON reports for detailed analysis and comparison

## Troubleshooting

### Common Issues

1. **Import Errors**: Make sure all dependencies are installed
2. **Permission Errors**: Run with appropriate permissions for system monitoring
3. **Memory Issues**: For long profiling sessions, monitor available memory
4. **Process Access**: Some system information may require elevated permissions

### Error Messages

- `PySide6 not available`: Install PySide6 for Qt application profiling
- `cProfile not available`: Built-in module should always be available
- `Permission denied`: Run with appropriate system permissions

## Integration

The profiler can be integrated into development workflows:

```bash
# Run profiler before making changes
python Tools/run_profiler.py --duration 300 --output-dir before_changes

# Make your changes...

# Run profiler after changes
python Tools/run_profiler.py --duration 300 --output-dir after_changes

# Compare the reports
```

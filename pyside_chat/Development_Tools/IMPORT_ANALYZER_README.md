# Import Analyzer Tool

A comprehensive tool for analyzing import statements in the PySide Chat codebase, detecting circular dependencies, and generating shared import files to reduce duplication.

## Features

- **📁 Codebase Scanning**: Analyzes all Python files in the pyside_chat directory
- **🔄 Circular Dependency Detection**: Identifies circular import dependencies
- **📊 Import Frequency Analysis**: Finds commonly used imports across the codebase
- **📋 Shared Import Generation**: Creates shared import files to reduce duplication
- **📄 Comprehensive Reporting**: Generates detailed JSON reports with recommendations
- **🔧 Import Optimization**: Suggests ways to optimize import statements

## Quick Start

### Basic Usage

```bash
# Run the analyzer with default settings
python pyside_chat/Development_Tools/run_import_analyzer.py

# Or run the analyzer directly
python pyside_chat/Development_Tools/import_analyzer.py
```

### Advanced Usage

```bash
# Analyze a different codebase path
python pyside_chat/Development_Tools/run_import_analyzer.py --codebase path/to/codebase

# Custom output directory for shared import files
python pyside_chat/Development_Tools/run_import_analyzer.py --output-dir custom/shared/imports

# Custom report file
python pyside_chat/Development_Tools/run_import_analyzer.py --report my_analysis.json

# Only generate shared import files for imports used in 5+ files
python pyside_chat/Development_Tools/run_import_analyzer.py --min-frequency 5

# Skip generating shared import files (analysis only)
python pyside_chat/Development_Tools/run_import_analyzer.py --no-shared-files

# Enable verbose output
python pyside_chat/Development_Tools/run_import_analyzer.py --verbose
```

## Usage Examples

### Using Generated Shared Import Files

After running the analyzer, you can use the generated shared import files:

```python

from PySide6.QtWidgets import QWidget, QVBoxLayout, QHBoxLayout

# Use the shared import file

# For logging
from pyside_chat.core.shared_imports.logging_imports import get_logger
logger = get_logger(__name__)
```

## Understanding the Analysis

### Circular Dependencies

The tool detects circular import dependencies like:

```
A -> B -> C -> A
```

This can cause import errors and should be resolved by:
- Moving shared code to a separate module
- Using dependency injection
- Restructuring the code architecture

### Shared Import Groups

The tool identifies commonly imported modules and groups them by category:

- **logging**: CustomLogger, LoggingHelpers
- **qt_core**: PySide6 Qt modules
- **typing**: Type hints and dataclasses
- **os_sys**: OS and system utilities
- **json_time**: JSON and time handling
- **threading**: Threading and concurrency
- **audio**: Audio processing libraries
- **ml_ai**: Machine learning libraries
- **requests**: HTTP and networking
- **numpy**: Numerical computing

### Import Frequency Analysis

The tool tracks how often each import is used:

```
PySide6.QtCore: 25 times
pyside_chat.core.logging.logger: 20 times
typing: 18 times
os: 15 times
```

## Command Line Options

| Option | Description | Default |
|--------|-------------|---------|
| `--codebase` | Path to codebase to analyze | `pyside_chat` |
| `--output-dir` | Output directory for shared imports | `pyside_chat/core/shared_imports` |
| `--report` | Output file for analysis report | `import_analysis_report.json` |
| `--min-frequency` | Minimum frequency for shared groups | `3` |
| `--no-shared-files` | Skip generating shared import files | `False` |
| `--verbose` | Enable verbose output | `False` |

## Integration with Development Workflow

### Pre-commit Hook

Add to your pre-commit hooks to check for circular dependencies:

```bash
#!/bin/bash
python pyside_chat/Development_Tools/run_import_analyzer.py --no-shared-files
if [ $? -ne 0 ]; then
    echo "Circular dependencies detected!"
    exit 1
fi
```

### CI/CD Integration

Add to your CI pipeline:

```yaml
- name: Analyze imports
  run: |
    python pyside_chat/Development_Tools/run_import_analyzer.py
    # Check if circular dependencies exist
    if grep -q "circular_dependencies" import_analysis_report.json; then
      echo "Circular dependencies found!"
      exit 1
    fi
```

## Troubleshooting

### Common Issues

1. **ImportError: No module named 'networkx'**
   ```bash
   pip install networkx
   ```

2. **Permission denied when creating shared import files**
   ```bash
   # Ensure the output directory exists and is writable
   mkdir -p pyside_chat/core/shared_imports
   chmod 755 pyside_chat/core/shared_imports
   ```

3. **Circular dependency detection not working**
   - Check that the codebase path is correct
   - Ensure all Python files are readable
   - Verify that relative imports are properly structured

### Debug Mode

Enable verbose output to see detailed analysis:

```bash
python pyside_chat/Development_Tools/run_import_analyzer.py --verbose
```
